## 📋 创建版本标签步骤

### 方法一：使用Git命令行（推荐）

#### 1. 确保所有更改已提交

```powershell
# 检查当前状态
git status

# 如果有未提交的更改，先提交
git add .
git commit -m "Release v1.0.0: 完整功能版本"
```

#### 2. 创建版本标签

```powershell
# 创建带注释的标签
git tag -a v1.0.0 -m "Release version 1.0.0

功能特性:
- 实时快讯推送系统
- 多渠道推送支持（Ghost/Telegram/Discord）
- 每日简报生成
- The Block Beats风格三栏布局
- 暗色主题界面
- 完整API和文档"
```

#### 3. 推送到GitHub

```powershell
# 推送标签到远程仓库
git push origin v1.0.0

# 如果推送所有标签
git push origin --tags
```

#### 4. 创建Release分支（可选但推荐）

```powershell
# 从当前提交创建release分支
git checkout -b release/v1.0.0

# 推送到远程
git push origin release/v1.0.0
```

### 方法二：使用GitHub Desktop

1. **提交所有更改**
   - 在GitHub Desktop中查看所有更改
   - 填写提交信息：`Release v1.0.0`
   - 点击"Commit to main"

2. **创建标签**
   - 点击"History"标签
   - 找到最新的提交
   - 右键点击 → "Create tag"
   - 标签名：`v1.0.0`
   - 描述：填写版本说明
   - 点击"Create tag"

3. **推送标签**
   - 点击"Push origin"推送标签

4. **在GitHub上创建Release**
   - 访问：https://github.com/q379078150-netizen/system-Webpage/releases/new
   - Tag: `v1.0.0`
   - Title: `v1.0.0 - 情报推送系统首个稳定版本`
   - Description: 复制 `CHANGELOG.md` 的内容
   - 点击"Publish release"

## 🎯 版本管理最佳实践

### 版本命名规则
- **主版本号**：重大功能更新
- **次版本号**：新功能添加
- **修订版本号**：Bug修复

例如：
- v1.0.0 - 首个稳定版本
- v1.1.0 - 添加新功能
- v1.0.1 - Bug修复

### 版本标签命名
- 使用 `v1.0.0` 格式
- 标签名与VERSION文件一致
- 创建Release时使用相同标签
