## 📋 步骤一：确保Git已安装

### 检查Git是否安装

打开PowerShell或命令提示符，运行：

```powershell
git --version
```

如果显示版本号（如 `git version 2.xx.x`），说明已安装。

如果显示错误，请先安装Git：

1. **方法一：使用winget（推荐）**
   ```powershell
   winget install Git.Git
   ```

2. **方法二：手动下载**
   - 访问：https://git-scm.com/download/win
   - 下载并安装
   - 安装时选择"Add Git to PATH"

3. **方法三：使用Chocolatey**
   ```powershell
   choco install git
   ```

安装完成后，**重新打开**PowerShell或命令提示符。

## 📋 步骤二：初始化Git仓库（如果未初始化）

### 检查是否已初始化


## 🔄 回退到v1.0.0的方法

### 方法一：创建回退分支（推荐）

```powershell
# 创建新分支指向v1.0.0
git checkout -b rollback-v1.0.0 v1.0.0

# 推送到远程
git push origin rollback-v1.0.0
```

### 方法二：切换到v1.0.0标签

```powershell
# 切换到v1.0.0标签（只读模式）
git checkout v1.0.0

# 查看代码后返回主分支
git checkout main
```

### 方法三：重置到v1.0.0（危险）

```powershell
# ⚠️ 警告：这会丢失v1.0.0之后的所有提交
git reset --hard v1.0.0

# 强制推送到远程（谨慎使用）
git push origin main --force
```

### 方法四：从GitHub下载

1. 访问：https://github.com/q379078150-netizen/system-Webpage/releases
2. 找到 v1.0.0 版本
3. 点击 "Source code (zip)" 下载
4. 解压并替换当前代码

## 🎯 版本管理最佳实践

### 版本命名规则
- **主版本号**：重大功能更新（如 v1.0.0 → v2.0.0）
- **次版本号**：新功能添加（如 v1.0.0 → v1.1.0）
- **修订版本号**：Bug修复（如 v1.0.0 → v1.0.1）

### 版本标签命名
- 使用 `v1.0.0` 格式
- 标签名与VERSION文件一致
- 创建Release时使用相同标签
